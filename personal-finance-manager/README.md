# Personal Finance Manager API

A comprehensive, production-ready REST API for personal finance management, built using Spring Boot, Spring Security, JPA, and an H2 database.

This project enables users to track income and expenses, organize transactions into built-in or custom categories, set and track savings goals with real-time percentage progress, and generate detailed monthly and yearly financial reports.

---

## 🌟 Key Features

- **User Management & Session Authentication**: Form-based authentication using Spring Security with HTTP session cookies. User registration validates unique email/usernames.
- **Transaction Tracking (Full CRUD)**: Record, filter, update, and delete transactions. Real-time validation checks for positive amounts and past/current transaction dates.
- **Category System**: Supports built-in standard categories (Salary, Food, Rent, Transportation, Entertainment, Healthcare, Utilities) alongside user-defined custom categories.
- **Savings Goals & Progress Computation**: Goals compute real-time progress (`currentProgress`, `percentage`, `remainingAmount`) dynamically derived from the user's net savings (income minus expense transactions) starting from the goal's `startDate`.
- **Advanced Reports & Financial Analytics**:
  - `Monthly Report`: Aggregates total income and total expenses grouped by category, returning net savings for any given month/year.
  - `Yearly Report`: Compiles yearly totals grouped by category to track financial behavior over the entire year.
- **Robust Exception Handling**: Global exception mapping using `@ControllerAdvice` to return standardized API responses for conflicts, bad requests, missing data, and validation errors.

## 📐 Design Decisions & Layered Architecture

The system is designed following the Clean layered architecture pattern:

```
[Client / HTTP Request]
         │
         ▼
[Controller Layer] (Authentication, DTO validation, Path/Request Parameter mapping)
         │
         ▼
[Service Layer] (Business logic, custom validations, report aggregations, progress calculation)
         │
         ▼
[Repository Layer] (Spring Data JPA interfaces executing database operations on H2 memory)
         │
         ▼
[H2 In-Memory Database]
```

### 1. Robust DTOs and API Contract
To decouple internal JPA entities from external API contracts, dedicated request/response DTOs are used:
- Optional fields for updates are handled via specialized DTOs (e.g. `TransactionUpdateRequest`, `GoalUpdateRequest`) allowing partial updates while preserving unchanged data.
- Global request validation is enforced using Jakarta validation annotations (e.g., `@NotBlank`, `@DecimalMin`, `@Pattern`).

### 2. Session-based Cookie Authentication
Form-based security context mapping is stored in the servlet session. State is managed transparently without heavy JWT setup, simplifying API clients (cookies are handled natively by browsers and HTTP clients). CSRF is disabled for easy client integration.

### 3. Data Isolation between Users
Strict multi-tenant isolation is enforced at the service layer:
- Users can only query, update, or delete transaction/goal records belonging to their authenticated session identifier.
- Attempting to access another user's record triggers an HTTP `404 Not Found` response.

## 🚀 Setup & Running Locally

### Prerequisites
- **Java JDK 17** or higher installed.
- **Maven** (configured in PATH, or use the Maven Wrapper `./mvnw` provided in the repository).

### Running the App
1. Clone or extract the project files.
2. Open a terminal and navigate to the project directory:
   ```bash
   cd c:\Users\darkh\Desktop\tnp
   ```
3. Start the application:
   ```bash
   # Windows Command Prompt / PowerShell
   .\mvnw.cmd spring-boot:run

   # Git Bash / Linux / macOS
   ./mvnw spring-boot:run
   ```
4. The API will start on `http://localhost:8080`.
5. Access the H2 Database console at `http://localhost:8080/h2-console` (JDBC URL: `jdbc:h2:mem:finance-db`, Username: `SA`, Password: `password` or empty).

## 📖 API Endpoint Reference

### 1. User Management & Authentication
| Method | Endpoint | Description | Request Body Example |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/auth/register` | Register a new account | `{"username": "user@example.com", "password": "Password123!", "fullName": "John Doe", "phoneNumber": "1234567890"}` |
| `POST` | `/api/auth/login` | Authenticate and start session | `{"username": "user@example.com", "password": "Password123!"}` |
| `POST` | `/api/auth/logout` | End current session | None |

### 2. Category Management
| Method | Endpoint | Description | Request Body Example / Response |
| :--- | :--- | :--- | :--- |
| `GET` | `/api/categories` | Get default and custom categories | Returns `{"categories": [...]}` |
| `POST` | `/api/categories` | Create custom category | `{"name": "Freelance", "type": "INCOME"}` |
| `DELETE` | `/api/categories/{name}` | Delete custom category | Returns success message (if unused) |

### 3. Transaction Management
| Method | Endpoint | Description | Query Parameters / Request Body |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/transactions` | Create a transaction | `{"amount": 1200.50, "date": "2024-05-24", "category": "Food", "description": "Grocery shopping"}` |
| `GET` | `/api/transactions` | Query transactions (sorted by date descending) | Query params (optional): `startDate`, `endDate`, `category` |
| `PUT` | `/api/transactions/{id}` | Update partial details | `{"amount": 1000.00, "description": "Discount applied"}` (date is immutable) |
| `DELETE` | `/api/transactions/{id}` | Delete transaction | None |

### 4. Savings Goals
| Method | Endpoint | Description | Request Body Example / Response |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/goals` | Create savings goal | `{"goalName": "Car Fund", "targetAmount": 15000, "targetDate": "2026-12-31", "startDate": "2024-01-01"}` |
| `GET` | `/api/goals` | List all goals with progress | Returns list with `currentProgress`, `progressPercentage`, `remainingAmount` |
| `GET` | `/api/goals/{id}` | Retrieve details of a goal | Returns detailed goal metrics |
| `PUT` | `/api/goals/{id}` | Update target details | `{"targetAmount": 18000, "goalName": "New Car Fund"}` |
| `DELETE` | `/api/goals/{id}` | Delete goal | None |

### 5. Financial Reports
| Method | Endpoint | Description | Response Details |
| :--- | :--- | :--- | :--- |
| `GET` | `/api/reports/monthly/{year}/{month}` | Monthly overview | Aggregates income, expenses by category, net savings |
| `GET` | `/api/reports/yearly/{year}` | Yearly overview | Year-wide breakdown of income, expenses, net savings |
